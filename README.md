This repository contains the P2P_chat.py module, which implements the peer-to-peer (P2P) chatting phase of a hybrid network chat application.
The logic in P2P_chat.py is divided into three main operational blocks:
1. The Receiving Phase (Peer as Server)The script registers specific HTTP POST routes using the AsynapRous framework to listen for incoming messages./send-peer: Handles direct, one-to-one messages from another peer in the network. It parses the incoming JSON payload to extract the sender's IP and message content./broadcast-peer: Handles broadcast messages sent to the general chat channel by other active peers.
2. The Sending Phase (Peer as Client)When sending messages, the peer acts as a client, establishing direct TCP connections to other peers without routing through a centralized server.send_p2p_message(): The core sending function. It opens a non-blocking connection to a target IP and Port using asyncio.open_connection(). It manually constructs a standard HTTP POST request containing a JSON payload and sends it directly to the target peer's AsynapRous endpoints.broadcast_message(): Iterates through the known active_peers list and uses asyncio.gather() to concurrently send the message to all connected peers, ensuring the event loop is not blocked.
3.  UI Integration Bridge/local-send: This API endpoint acts as the bridge between the frontend GUI (JavaScript) and the backend logic. When a user hits "Send" on the interface, the frontend sends a POST request here. The backend then extracts the message type (broadcast or direct) and pushes the corresponding asynchronous sending tasks (broadcast_message or send_p2p_message) into the asyncio event loop.
